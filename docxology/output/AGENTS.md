# docxology/output — Diagnostic Artifacts Agentic Contract

## 0. Purpose
Machine-optimized index for the `output` hierarchy. This directory is the persistent store for all high-fidelity diagnostics, tensor traces, and visual reports generated by the docxology sidecar.

## 1. Hierarchy Registry

| Category | Primary Workloads | Key Artifacts |
| :--- | :--- | :--- |
| **`advanced/`** | Neural encoders, complex dependencies. | `*_beliefs.png`, `*_model_trace.npz` |
| **`api/`** | Model construction smoke tests. | `*_validation.json`, `*_full_data.json` |
| **`docxology/`** | Sidecar reflection and internal benchmarks. | `*_execution_report.md` |
| **`envs/`** | Active Inference Environments. | `*_actions.png`, `*_vfe_traj.png` |
| **`experimental/`** | SI and MCTS planning traces. | `*_efe_heatmap.png`, `*_pi_tree.json` |
| **`inference_and_learning/`** | FPI, MMP, VMP convergence. | `*_vfe_convergence.png`, `*_qs_error.json` |
| **`legacy/`** | NumPy parity and regression checks. | `*_parity_report.md` |
| **`sparse/`** | BCOO efficiency and JIT benchmarks. | `*_mem_usage.png`, `*_sparse_dot.json` |

## 2. Invariant Constraints
Agents interacting with these artifacts MUST adhere to the following rules:
- **Immutable Traces**: Once generated by `run_all.py`, NPZ and JSON files must be treated as the ground-truth numerical state of the simulation for that version of `pymdp`.
- **Automated Management**: This directory is managed exclusively by the thin orchestrator implementation. manual additions are permitted but will not be indexed by the validation matrix.
- **Relative Linking**: All auto-generated Markdown reports must use relative paths to images to ensure portability across local and hosted repository viewers.

## 3. Deployment Notes
This directory is often excluded from version control for high-frequency CI runs but serves as the definitive source of thermodynamic confirmation for "verified" releases.
