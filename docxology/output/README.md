# docxology/output

> **Validation Artifacts & Diagnostic Traces**

This directory is the persistent store for all artifacts generated by the [thin orchestrator](../pkg/support/mirror_dispatch.py). Every execution target registered in the sidecar produces a standardized set of high-fidelity diagnostics organized by category.

## 📂 Output Hierarchy

| Category | Description |
| :--- | :--- |
| **`advanced/`** | Deep action-dependencies and complex neural-encoder mappings. |
| **`api/`** | Basic model construction and `pymdp` API surface smoke tests. |
| **`docxology/`** | Internal sidecar benchmarks and minimal discrete-timestep validations. |
| **`envs/`** | Curated [Active Inference environments](https://github.com/infer-actively/pymdp/tree/main/examples/envs). |
| **`experimental/`** | Sophisticated Inference (SI) and Monte Carlo Tree Search (MCTS) traces. |
| **`inference_and_learning/`** | Comparative traces for FPI, MMP, and VMP state inference. |
| **`legacy/`** | Parity checks against original `pymdp` GridWorld and T-Maze demos. |
| **`sparse/`** | Performance and accuracy benchmarks for sparse tensor backends. |

## 📦 Artifact Matrix (Per Scenario)

Each execution of `run_all.py` generates four core artifacts for every target, identified by the script `{stem}`:

| File Pattern | Format | content |
| :--- | :--- | :--- |
| **`{stem}_validation.json`** | JSON | High-level success flags and handler-specific diagnostics. |
| **`{stem}_full_data.json`** | JSON | Complete `info` dictionary including retroactive Shannon entropy and invariants. |
| **`{stem}_model_trace.npz`** | NPZ | Unrestricted mathematical trace of all extracted tensors for NumPy/JAX analysis. |
| **`{stem}_execution_report.md`** | MD | Auto-generated report with Performance Insights (Entropy, VFE, EFE, KL) and embedded visuals. |

## 🖼️ Visual Diagnostics

Depending on the data available, the orchestrator triggers up to 13 visualization types:
- **Belief Heatmaps (`_beliefs.png`)**: $q(s)$ posterior over time.
- **Action Probabilities (`_actions.png`)**: Distribution over control factors.
- **Negative EFE (`_efe_heatmap.png`)**: Expected Free Energy $G$ across policies and horizons.
- **Free Energy Trajectories (`_efe_traj.png`)**: Variational Free Energy $F$ convergence.

---

## 🤖 Agentic Contract

- **Automated Management**: This directory is managed exclusively by `run_all.py`. Manual deletions are permitted to reset benchmarks, but manual additions will be ignored by the validation matrix.
- **Path Resolution**: References in the `execution_report.md` use relative paths to ensure portability within the repository.
